# 情绪标签管理 API 文档

> 版本：v1（2026-05-06）
> 适用：心理健康系统 backend 模块
> 路径前缀：`/api/mood-tags`
> 鉴权：所有接口要求请求头 `Authorization: Bearer <JWT>`，标签数据按用户隔离，用户只能操作自己的标签。

---

## 1. 通用约定

### 1.1 请求/响应格式

- 所有接口均为 JSON（`Content-Type: application/json`），统一返回如下结构：

```json
{
  "code": 200,
  "msg": "success",
  "data": <T>
}
```

- 业务错误：`code != 200`，`msg` 为提示文案；常见 `code`：
  - 400：参数/业务校验失败（如标签名重复、名称为空等）
  - 401：未登录或 token 失效
  - 403：尝试操作他人标签
  - 404：标签不存在

### 1.2 数据库迁移

新增表与字段见 `backend/src/main/resources/db/migration/V20260506__mood_tags_refactor.sql`，需手工执行：

- 新增 `mood_tag` 表（用户自定义情绪标签）
- `mood_diary` 表新增 `tag_id` 外键列

## 2. 标签 CRUD 接口

### 2.1 获取当前用户的所有标签

`GET /api/mood-tags`

无请求参数，返回当前登录用户的全部标签列表（按创建时间升序）。

**响应示例：**

```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "id": 1,
      "name": "开心",
      "color": "#67C23A",
      "createdAt": "2026-05-06 10:30:00"
    },
    {
      "id": 2,
      "name": "焦虑",
      "color": "#E6A23C",
      "createdAt": "2026-05-06 10:31:00"
    },
    {
      "id": 3,
      "name": "低落",
      "color": "#909399",
      "createdAt": "2026-05-06 10:32:00"
    }
  ]
}
```

### 2.2 创建标签

`POST /api/mood-tags`

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| name | string | 是 | 标签名称，最长 50 字符，同一用户下不可重复 |
| color | string | 否 | 十六进制颜色值，格式 `#RRGGBB`（如 `#FF5733`）；不传则默认 `#409EFF` |

**请求示例：**

```json
{
  "name": "开心",
  "color": "#67C23A"
}
```

**响应示例：**

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": 1,
    "name": "开心",
    "color": "#67C23A",
    "createdAt": "2026-05-06 10:30:00"
  }
}
```

**错误情况：**

| code | msg | 触发条件 |
| --- | --- | --- |
| 400 | 标签名称不能为空 | name 为空或纯空白 |
| 400 | 标签名称长度不能超过 50 个字符 | name 超长 |
| 400 | 颜色格式必须为十六进制，如 #FF5733 | color 格式不合法 |
| 400 | 标签名称已存在 | 当前用户已有同名标签 |

---

### 2.3 更新标签

`PUT /api/mood-tags/{id}`

**路径参数：**

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | long | 标签 ID |

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| name | string | 是 | 新的标签名称 |
| color | string | 否 | 新的颜色值；不传则保持原色 |

**请求示例：**

```json
{
  "name": "非常开心",
  "color": "#55D187"
}
```

**响应示例：**

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": 1,
    "name": "非常开心",
    "color": "#55D187",
    "createdAt": "2026-05-06 10:30:00"
  }
}
```

**错误情况：**

| code | msg | 触发条件 |
| --- | --- | --- |
| 400 | 标签名称已存在 | 改名后与已有标签冲突 |
| 403 | 无权限操作他人标签 | 尝试更新不属于自己的标签 |
| 404 | 标签不存在 | id 无效 |

---

### 2.4 删除标签

`DELETE /api/mood-tags/{id}`

**路径参数：**

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | long | 标签 ID |

**响应示例：**

```json
{
  "code": 200,
  "msg": "success",
  "data": "删除成功"
}
```

**错误情况：**

| code | msg | 触发条件 |
| --- | --- | --- |
| 400 | 该标签下存在 N 篇日记，请先解除关联后再删除 | 标签仍被日记引用 |
| 403 | 无权限操作他人标签 | 尝试删除不属于自己的标签 |
| 404 | 标签不存在 | id 无效 |

## 3. 日记接口变更说明

标签重构后，情绪日记的创建/更新/查询接口有以下变化：

### 3.1 创建日记

`POST /api/diary`

**请求体变更：**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| content | string | 是 | 日记内容，最长 500 字符 |
| moodScore | integer | 是 | 心情分值 1-10 |
| ~~tags~~ | ~~string[]~~ | - | **已废弃，不再使用** |
| tagId | long | 否 | 关联的情绪标签 ID（需为当前用户自己的标签） |

**请求示例：**

```json
{
  "content": "今天阳光很好，心情不错",
  "moodScore": 8,
  "tagId": 1
}
```

### 3.2 更新日记

`PUT /api/diary/{id}`

**请求体变更：**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| content | string | 是 | 日记内容 |
| moodScore | integer | 是 | 心情分值 1-10 |
| ~~tags~~ | ~~string[]~~ | - | **已废弃，不再使用** |
| tagId | long | 否 | 关联标签 ID；传 `null` 表示取消关联 |

### 3.3 分页查询

`GET /api/diary/page`

**新增查询参数：**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| tagId | long | 否 | 按标签筛选日记 |

其他参数（page、size、startDate、endDate、minScore、maxScore）不变。

### 3.4 日记响应结构变更

旧的 `tags: string[]` 字段替换为 `tag` 对象：

```json
{
  "id": 100,
  "date": "2026-05-06 10:30",
  "content": "今天阳光很好，心情不错",
  "score": 8,
  "tag": {
    "id": 1,
    "name": "开心",
    "color": "#67C23A"
  },
  "aiAnalysis": "...",
  "createdAt": "2026-05-06 10:30:00"
}
```

> 若日记未关联任何标签，`tag` 字段为 `null`。

## 4. 前端集成建议

### 4.1 标签管理页面

- 用户进入「标签管理」或「写日记」页面时，调用 `GET /api/mood-tags` 获取可用标签列表
- 标签以 `color` 指定的颜色作为背景/边框色渲染（Tag / Badge 组件）
- 新增/编辑标签时可提供颜色选择器（预设常用色 + 自定义输入）

### 4.2 写日记流程

1. 页面加载时拉取标签列表
2. 用户选择一个标签（可选），记录其 `id`
3. 提交时将 `tagId` 放入请求体

### 4.3 日记列表

- 列表中每条日记的 `tag` 对象可直接渲染为带颜色的标签徽章
- 支持点击标签快速筛选（将 `tagId` 传入分页查询参数）

### 4.4 推荐预设颜色

```javascript
const PRESET_COLORS = [
  '#67C23A', // 绿色 - 开心/平静
  '#409EFF', // 蓝色 - 默认
  '#E6A23C', // 橙色 - 焦虑/紧张
  '#F56C6C', // 红色 - 愤怒/烦躁
  '#909399', // 灰色 - 低落/疲惫
  '#9B59B6', // 紫色 - 忧郁
  '#1ABC9C', // 青绿 - 放松
  '#F39C12', // 金色 - 兴奋
]
```
